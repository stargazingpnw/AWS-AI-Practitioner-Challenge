# AWS AI Practitioner Challenge - PartyRock Portfolio

Welcome to my project portfolio for the **AWS AI & ML Scholars / AI Practitioner Challenge**. This repository showcases the interactive, AI-powered applications and analytical tools I built using **Amazon PartyRock** and **Amazon Bedrock** foundational models.

The projects in this portfolio demonstrate core competencies in Generative AI, foundational model orchestration, strict safety constraint engineering, and user-centric workflow automation.

---

## 📂 Portfolio Projects

### 🧠 [Project 1: NutriWellness Personal Health Coach](./project-1-nutriwellness/)
*   **Core Focus:** Health, Nutrition, and Mindset Alignment.
*   **Description:** A single-session "Morning Routine" coach that transforms yesterday's food logs and steps, baseline physical metrics, and today's subjective morning mood into bite-sized, executable nutrition and fitness targets. Features strict guardrail enforcement for 6 distinct dietary styles and open-text food allergy filters to prevent cognitive overload.
*   **Technologies Used:** Amazon PartyRock, Amazon Bedrock, Prompt Constraint Engineering, Custom Input/Output UI Widgets.
*   **[👉 View Full Project Details & Readme](./project-1-nutriwellness/README.md)**

### 📊 [Project 2: College Cohort Gut Microbiome & Food Insecurity Analysis](./project-2-data-analysis/)
*   **Core Focus:** Foundational Model Data Analysis & Biological Trend Synthesis.
*   **Description:** A data investigation project using PartyRock’s *Analyze Data* workflow to parse complex biological datasets. The project explores the relationship between food insecurity and gut microbiome/metabolome variations among college students using multi-turn prompting and AI-generated analysis tables.
*   **Technologies Used:** PartyRock Data Analyzer, Prompt-Driven Synthesis, Metabolomic Evaluation.
*   **[👉 View Full Project Details & Readme](./project-2-data-analysis/README.md)**

## 🧠 Core Competencies Demonstrated Across Both Projects
*   **Prompt Engineering & Constraint Design:** Writing rigid instructions to control AI output behavior, enforce formatting boundaries, and completely eliminate model hallucinations.
*   **Architecture Pivoting:** Navigating no-code/low-code platform constraints (like single-session state limits) and successfully redesigning application logic to still deliver high-utility tools.
*   **Responsible AI Integration:** Handling sensitive user data safely by focusing on immediate advisory outputs, blocking personal contact inputs (like names/emails), and prioritizing critical safety guardrails (like allergy checks).
