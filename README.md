# AWS AI Practitioner Challenge - PartyRock Portfolio
### Integrated Multi-Omic Product & Behavioral Analytics Platform

Welcome to my project portfolio for the **AWS AI & ML Scholars / AI Practitioner Challenge**. This repository showcases interactive, AI-powered applications and deep data investigation pipelines built using **Amazon PartyRock** and **Amazon Bedrock** foundational models.

The projects in this portfolio demonstrate core competencies in Generative AI, foundational model orchestration, strict safety constraint engineering, and user-centric workflow automation.

### 💡 Cross-Functional Product Vision: The Gut-Brain Connect
Inspired by the clinical research frameworks in [*The Psychobiotic Revolution: Mood, Food, and the New Science of the Gut-Brain Connection*](https://amazon.com), this portfolio evaluates how lifestyle dynamics, digital habits, and metabolic stressors alter peripheral chemistry. 

By analyzing user-facing tracking telemetry (Project 1) alongside a separate, peer-reviewed clinical mass-spectrometry dataset (Project 2), this workspace bridges the gap between consumer digital health and backend bioinformatic data engineering. It demonstrates a scalable, regulatory-aware blueprint for **personalized healthtech platforms, digital therapeutics (DTx), and translational clinical data management.**

## 📂 Portfolio Projects

### 🏃 [Project 1: NutriWellness Personal Health Coach](./project-1-nutriwellness/)
*   **Core Focus:** Health, Nutrition, and Mindset Alignment.
*   **Module Type:** Digital Health Tracking & Telemetry Layer (Cohort 1)
*   **Description:** A single-session application that acts as a "Morning Routine" coach. It ingests yesterday's user food logs, steps, baseline physical metrics, and a subjective morning mood score to synthesize immediate, executable nutrition and fitness targets. 
*   **TPM & Technical Highlights:** Designed strict input/output UI widget structures and rigorous prompt guardrails enforcing 6 distinct dietary styles. Implemented open-text allergen filters to prevent hazardous outputs and eliminate cognitive overload for the end user.
*   **Technologies Used:** Amazon PartyRock, Amazon Bedrock, Prompt Constraint Engineering, Telemetry Validation UI Widgets.
*   **[👉 View Full Project Details & Readme](./project-1-nutriwellness/README.md)**

### 🧪 [Project 2: College Cohort Nutrition & Metabolomic Profile Exploration](./project-2-data-analysis/)
*   **Core Focus:** Foundational Model Data Analysis & Biological Trend Synthesis.
*   **Module Type:** Clinical Informatics & Data Integration Layer (Cohort 2)
*   **Description:** A data investigation module leveraging PartyRock’s *Analyze Data* workflow to parse complex biological datasets. The engine explores the relationship between nutrient profiles, lifestyle habits (dorm status, screen time), and mass-spectrometry serum metabolome variations among college students.
*   **TPM & Technical Highlights:** Utilized an independent, peer-reviewed clinical dataset sourced directly from [Mendeley Data (Mohr et al., n=60)](https://mendeley.com) to analyze serum and fecal metabolomic profiles under nutritional constraints. Orchestrated multi-turn analytical prompting to build integrated physiological data matrix schemas.
*   **Technologies Used:** PartyRock Data Analyzer, Prompt-Driven Bioinformatic Synthesis, Path-Analysis Evaluation Matrix.

## 🧠 Core Competencies Demonstrated Across Both Projects
*   **Prompt Engineering & Constraint Design:** Writing rigid instructions to control AI output behavior, enforce formatting boundaries, and completely eliminate model hallucinations.
*   **Architecture Pivoting:** Navigating no-code/low-code platform constraints (like single-session state limits) and successfully redesigning application logic to still deliver high-utility tools.
*   **Responsible AI Integration:** Handling sensitive user data safely by focusing on immediate advisory outputs, blocking personal contact inputs (like names/emails), and prioritizing critical safety guardrails (like allergy checks).
